# 故事协议 V2

本文档只描述 **V2 协议**。

它是一套独立的故事数据协议，不依赖 V1，也不把 V1 当成自己的前置语法来描述。

---

## 1. 协议定位

V2 是当前主协议。

特点：

- 顶层是 **显式文档对象**
- 使用 `schemaId` 作为强识别标记
- 使用 `nodes[]` 存储节点数组
- 使用 `content[]` 存储结构化内容 token
- 使用 `target` 表示分支跳转目标
- 可以额外携带编辑器布局信息

适合：

- 稳定的结构化数据存储
- 大模型生成
- 外部工具处理
- 更强的字段校验
- 编辑器导入导出

---

## 2. 顶层结构

V2 顶层必须是一个对象。

标准结构示例：

```json
{
  "schemaId": "cassette-if.story/v2",
  "format": "cassette-if",
  "version": 2,
  "title": "故事标题",
  "startNodeId": "start",
  "nodes": [
    {
      "id": "start",
      "image": "images/example.jpg",
      "content": [
        { "type": "text", "value": "欢迎来到这里。" }
      ],
      "options": [
        { "text": "继续", "target": "end" }
      ]
    }
  ],
  "editor": {
    "nodePositions": {
      "start": { "x": 100, "y": 100 }
    }
  }
}
```

---

## 3. 顶层字段定义

### `schemaId`
类型：`string`  
是否必填：是

固定值：

```json
"cassette-if.story/v2"
```

作用：

- 作为 V2 协议的强识别标记
- 用于和其他协议明确区分

---

### `format`
类型：`string`  
是否必填：是

当前固定值：

```json
"cassette-if"
```

作用：

- 标识故事数据所属的协议族

---

### `version`
类型：`number`  
是否必填：是

当前固定值：

```json
2
```

作用：

- 标识该文档使用的协议版本号

---

### `title`
类型：`string`  
是否必填：是

作用：

- 故事标题

---

### `startNodeId`
类型：`string`  
是否必填：是

作用：

- 指定入口节点 ID

要求：

- 必须指向 `nodes[]` 中某个已存在节点的 `id`

---

### `nodes`
类型：`array`  
是否必填：是

作用：

- 存放所有故事节点

要求：

- 数组不能为空
- 每个节点必须具有唯一 `id`

---

### `editor`
类型：`object`  
是否必填：否

作用：

- 存放仅供编辑器使用的布局信息
- 不参与故事逻辑本身

当前常见结构：

```json
"editor": {
  "nodePositions": {
    "start": { "x": 100, "y": 100 }
  }
}
```

---

## 4. 节点对象结构

每个 `nodes[]` 项支持以下字段：

### `id`
类型：`string`  
是否必填：是

作用：

- 节点唯一标识

---

### `image`
类型：`string`  
是否必填：否

作用：

- 节点图片路径

示例：

```json
"image": "images/塞小林.jpg"
```

---

### `content`
类型：`array`  
是否必填：是

作用：

- 按顺序描述节点正文内容
- 使用结构化 token，而不是单个脚本文本字符串

---

### `options`
类型：`array`  
是否必填：是

作用：

- 定义节点选项

---

## 5. 选项对象结构

每个 `options[]` 项支持：

### `text`
类型：`string`  
是否必填：是

作用：

- 选项按钮显示文本

### `target`
类型：`string`  
是否必填：是（建议）

作用：

- 指向目标节点 ID

---

## 6. content token 协议

`content[]` 是 V2 的核心。

每个 token 是一个对象，当前支持以下三类。

---

### 6.1 `text` token

结构：

```json
{ "type": "text", "value": "普通文本" }
```

字段：

- `type`: 固定为 `text`
- `value`: 字符串内容

---

### 6.2 `pause` token

结构：

```json
{ "type": "pause", "ms": 500 }
```

字段：

- `type`: 固定为 `pause`
- `ms`: 停顿毫秒数

---

### 6.3 `styled` token

结构：

```json
{
  "type": "styled",
  "color": "#0f0",
  "underline": false,
  "strike": false,
  "children": [
    { "type": "text", "value": "成功" }
  ]
}
```

字段：

- `type`: 固定为 `styled`
- `color`: 字符串，可为空
- `underline`: 布尔值
- `strike`: 布尔值
- `children`: 子 token 数组

说明：

- `children` 用于嵌套内容
- `styled` 可以包裹 `text`、`pause`、甚至其他 `styled`
- 因此 V2 支持结构化嵌套样式

---

## 7. 结构约束

建议满足以下约束：

1. `schemaId` 必须是 `cassette-if.story/v2`
2. `format` 必须是 `cassette-if`
3. `version` 必须是 `2`
4. `startNodeId` 必须存在于 `nodes[].id`
5. `nodes[].id` 必须全局唯一
6. `options[].target` 应该指向有效节点 ID
7. `content` 应该是数组，不建议为 `null`
8. `editor` 可以缺省

---

## 8. 最小示例

```json
{
  "schemaId": "cassette-if.story/v2",
  "format": "cassette-if",
  "version": 2,
  "title": "最小示例",
  "startNodeId": "start",
  "nodes": [
    {
      "id": "start",
      "content": [
        { "type": "text", "value": "欢迎。" }
      ],
      "options": [
        { "text": "结束", "target": "end" }
      ]
    },
    {
      "id": "end",
      "content": [
        { "type": "text", "value": "结束。" }
      ],
      "options": []
    }
  ]
}
```

---

## 9. 推荐使用场景

建议使用 V2 的情况：

- 新建故事
- 需要给大模型生成
- 需要结构化编辑
- 需要长期维护
- 需要更强的字段校验
- 需要与编辑器和工具链稳定协作

如果数据只要求兼容旧项目而不追求结构化，可以选择另一套独立协议，而不是在 V2 中回退到旧语法。

# 故事协议 V1

本文档只描述 **V1 协议**。

它是一套独立的故事数据协议，不依赖 V2，也不把 V2 作为自己的后继版本来描述。

---

## 1. 协议定位

V1 是项目早期使用的故事协议。

特点：

- 顶层是 **节点字典对象**
- 每个节点用一个唯一 key 标识
- 节点正文使用 **单个 `text` 字符串**
- 文本样式与停顿通过 **内联脚本标记** 表达
- 分支跳转通过 `nextNode` 表示

适合：

- 手工快速编写
- 旧版本项目兼容
- 文本为主、结构约束较弱的场景

不适合：

- 给大模型稳定生成
- 做强校验
- 做结构化编辑

---

## 2. 顶层结构

V1 顶层必须是一个对象。

每个顶层 key 表示一个节点 ID，每个 value 表示该节点内容。

示例：

```json
{
  "start": {
    "image": "images/example.jpg",
    "text": "系统初始化中...\\.",
    "options": [
      {
        "text": "继续",
        "nextNode": "node_2"
      }
    ]
  },
  "node_2": {
    "text": "这里是第二个节点。",
    "options": []
  }
}
```

---

## 3. 顶层规则

### 必须满足

1. 顶层必须是 JSON 对象
2. 顶层每个 key 必须是唯一节点 ID
3. 顶层每个 value 必须是节点对象

### 推荐约定

- 使用 `start` 作为入口节点 ID
- 节点 ID 使用稳定、可读的字符串

### 注意事项

- **不要在 V1 顶层增加额外元数据字段**（例如 `schemaId`、`version`、`meta`、`_magic`）
- 因为旧版读取逻辑通常会把顶层所有 key 都当成节点处理

不安全示例：

```json
{
  "schemaId": "...",
  "start": { ... }
}
```

---

## 4. 节点对象结构

每个节点对象支持以下字段。

### `image`
类型：`string`  
是否必填：否

作用：

- 指定节点对应图片路径

示例：

```json
"image": "images/example.jpg"
```

---

### `text`
类型：`string`  
是否必填：是

作用：

- 保存节点正文
- 正文中允许嵌入 V1 的文本控制标记

示例：

```json
"text": "系统初始化中...\\.\\n欢迎来到 \\C[#0f0]{第七扇区}。"
```

---

### `options`
类型：`array`  
是否必填：是

作用：

- 定义该节点的分支选项

示例：

```json
"options": [
  {
    "text": "检查日志",
    "nextNode": "logs"
  }
]
```

---

## 5. 选项对象结构

每个 `options[]` 项支持以下字段：

### `text`
类型：`string`  
是否必填：是

作用：

- 按钮上显示的文本

### `nextNode`
类型：`string`  
是否必填：是（建议）

作用：

- 指向下一个节点 ID

---

## 6. 文本标记语法

V1 的正文样式和节奏控制都写在 `text` 字符串中。

### 6.1 停顿

语法：

```text
\.
```

含义：

- 插入一次固定停顿
- 当前实现中通常对应 **500ms**

示例：

```text
系统初始化中...\.
```

---

### 6.2 颜色文本

语法：

```text
\C[color]{内容}
```

示例：

```text
\C[#0f0]{通过}
\C[red]{危险}
```

含义：

- 将花括号中的内容按指定颜色显示

---

### 6.3 下划线文本

语法：

```text
\_{内容}
```

示例：

```text
\_{深界监听站 Delta-12}
```

含义：

- 将内容以下划线形式显示

---

### 6.4 删除线文本

语法：

```text
\-{内容}
```

示例：

```text
\-{协议已被打破}
```

含义：

- 将内容以删除线形式显示

---

## 7. 结构约束

建议满足以下约束：

1. 节点 ID 全局唯一
2. `options[].nextNode` 应该指向存在的节点 ID
3. `options` 可为空数组，表示结局或无分支节点
4. `text` 应为字符串，不建议为 `null`

---

## 8. 最小示例

```json
{
  "start": {
    "text": "欢迎进入系统。",
    "options": [
      {
        "text": "继续",
        "nextNode": "end"
      }
    ]
  },
  "end": {
    "text": "结束。",
    "options": []
  }
}
```

---

## 9. 推荐使用场景

建议使用 V1 的情况：

- 需要兼容旧项目
- 临时快速手工写一个小故事
- 已经大量依赖 `text` 内联标记

如果需要更强结构化、更稳定地让工具或大模型处理，建议改用另一套独立协议，而不是继续扩展 V1 本身。

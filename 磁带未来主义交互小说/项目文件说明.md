# 项目文件说明

本文档用于解释当前项目的目录结构、文件作用与页面之间的关系，方便第一次接触项目的人快速理解代码组织方式。

---

## 1. 项目整体定位

这是一个基于单页 HTML + 外部共享 CSS/JS 的交互小说项目，主要包含三类内容：

1. **运行页**：负责播放故事、显示文本、图片与选项。
2. **编辑器**：负责可视化编辑故事节点并导出 JSON。
3. **故事数据**：使用 V1 / V2 协议描述剧情节点、分支与图片。

项目当前已经完成：

- 运行时 JS 模块化
- 运行时 CSS 共享化
- 故事协议从旧版 V1 迁移到新版 V2
- 对旧版 V1 数据的兼容读取
- 故事文件集中到 `stories/`
- 图片资源集中到 `images/`

## 3. 三个运行页

### 3.1 `磁带主义-打开本地文件版本.html`

作用：

- 当前最完整的运行页
- 支持：
  - 自动探测 `stories/磁带主义story.json`
  - 手动选择本地 JSON 文件
  - 图片显示
  - CRT 风格文本播放
- 适合作为主运行入口

它现在本身只负责：

- 页面 DOM 结构
- 引入共享 CSS
- 引入页面入口 JS：`js/pages/runtime-open-local.js`

---

### 3.2 `磁带主义-读取本地文件版本.html`

作用：

- 一个更简单的运行页
- 直接通过 `fetch('./stories/磁带主义story.json')` 读取故事
- 适合本地静态服务器场景

它本身只负责：

- 页面 DOM 结构
- 引入共享 CSS
- 引入页面入口 JS：`js/pages/runtime-fetch.js`

---

### 3.3 `磁带未来主义交互小说页面.html`

作用：

- 内联故事示例页
- 页面内直接放置 `window.__INLINE_STORY__`
- 不依赖外部 JSON 文件即可演示运行效果

它本身只负责：

- 页面 DOM 结构
- 注入一个内联示例故事对象
- 引入共享 CSS
- 引入页面入口 JS：`js/pages/runtime-inline.js`

---

## 4. 编辑器页面

### `磁带主义剧本编辑器.html`

作用：

- 可视化节点编辑器
- 支持拖拽节点、编辑选项跳转、预览图片路径
- 可导入 V1 / V2 故事文件
- 导出统一的 V2 协议 JSON

它本身只负责：

- 编辑器 DOM 结构
- 页面内的工具栏、节点容器和上下文菜单
- 引入编辑器入口 JS：`js/pages/story-editor.js`

说明：

- 编辑器内部编辑时仍然使用一种“便于文本输入”的中间状态
- 导出时会自动转换为标准 V2 格式
- 本地图片路径默认写为 `images/<filename>`

---

## 5. 说明页

### `介绍页.html`

作用：

- 项目展示页 / 说明页
- 现在已经更新为 V2 故事协议说明页面
- 里面包含：
  - 顶层字段解释
  - `content` token 结构解释
  - `options.target` 的含义
  - 从 V1 到 V2 的迁移说明

如果别人要理解当前故事 JSON 该怎么写，这个页面是最直观的入口之一。

---

## 6. 数据目录

### 6.1 `stories/`

作用：

- 统一存放所有故事 JSON
- 当前已集中存放：
  - `stories/磁带主义story.json`
  - `stories/磁带主义story.legacy.json`
  - `stories/mygo.json`
  - `stories/巡逻舰经过时_story.v2.json`

说明：

- 运行页默认读取的是 `stories/磁带主义story.json`
- 编辑器也推荐从这个目录中导入/导出故事文件

### 6.2 `images/`

作用：

- 统一存放本地图片资源
- 当前已集中存放：
  - `images/塞小林.jpg`
  - `images/GGPDq5XbsAEeCwZ.jpg`

说明：

- 故事 JSON 中的本地图片路径统一推荐写成 `images/...`
- 运行时不会猜测路径，而是直接按 JSON 中的 `image` 字段加载

---

## 7. 主故事与样例故事文件

### 7.1 `stories/磁带主义story.json`

作用：

- 当前主故事文件
- 使用 **V2 协议**
- 是运行页默认读取的主数据源

重要字段：

- `schemaId`: 用于强识别 V2 版本
- `format`
- `version`
- `title`
- `startNodeId`
- `nodes`
- `editor.nodePositions`

这是当前项目最重要的数据文件。

---

### 7.2 `stories/磁带主义story.legacy.json`

作用：

- 主故事文件的旧版备份
- 保留 V1 结构，方便对照迁移结果
- 也可作为兼容测试样例

它不是当前默认运行文件，但很适合排查 V1 / V2 兼容问题。

---

### 7.3 `stories/mygo.json`

作用：

- 另一份旧版 V1 样例故事
- 主要用于测试旧协议兼容性

说明：

- 该文件引用的是 `images/...` 路径
- 若对应图片文件不存在，会出现图片 404
- 这属于资源缺失，不是协议或运行时错误

---

### 7.4 `stories/巡逻舰经过时_story.v2.json`

作用：

- 另一份 V2 故事文件
- 已从旧的 `小说/` 目录语义中抽离为统一故事数据文件

说明：

- `小说/` 目录现在更适合保留原始文字脚本，而不是作为运行时故事 JSON 主目录

## 8. CSS 目录

目录：`css/runtime/`

这是三个运行页共享的视觉样式系统，统一采用“打开本地文件版本”的风格。

### 8.1 `css/runtime/index.css`

作用：

- 运行时 CSS 的统一入口文件
- 负责聚合其他样式文件
- 三个运行页都只需要引用这一份

---

### 8.2 `css/runtime/tokens.css`

作用：

- 定义运行页主题变量
- 例如：主色、背景色、CRT 阴影等

适合修改场景：

- 全局改色
- 调整整体氛围

---

### 8.3 `css/runtime/base.css`

作用：

- 定义页面级基础环境
- 如 `body` 的布局、背景、字体、对齐方式等

---

### 8.4 `css/runtime/shell.css`

作用：

- 定义 CRT 外壳与主要结构布局
- 包括：
  - `.crt-container`
  - `.screen`
  - `.header`
  - `.tape-label`

---

### 8.5 `css/runtime/effects.css`

作用：

- 定义 CRT 特效层
- 包括：
  - 扫描线
  - 扫描光束
  - 闪烁层
  - 暗角
  - 对应动画 keyframes

---

### 8.6 `css/runtime/content.css`

作用：

- 定义正文内容区与滚动条、光标样式
- 包括：
  - `#story-text`
  - `.char-span`
  - `.cursor::after`
  - 滚动条样式

---

### 8.7 `css/runtime/controls.css`

作用：

- 定义选项按钮和按钮 hover 效果
- 包括：
  - `#choices`
  - `button`
  - `button:hover`
  - `button::before`

---

### 8.8 `css/runtime/image-panel.css`

作用：

- 定义运行页里的图片显示样式
- 包括：
  - `.terminal-img-container`
  - `.terminal-img`
  - `.terminal-img-overlay`
  - `img-scan-in` 动画

说明：

- 这个文件来自“打开本地文件版本”的样式增强
- 现在已经成为三个运行页统一共享的图片视觉标准

## 9. JS 目录

目录：`js/`

分为两层：

- `js/runtime/`：共享运行时逻辑
- `js/pages/`：页面级入口逻辑

---

## 10. `js/runtime/`：共享运行时模块

### 10.1 `js/runtime/formats/story-format-v1.js`

作用：

- 专门处理旧版 V1 故事协议
- 负责：
  - 旧版文本标记解析
  - `nextNode` -> 统一结构转换
  - V1 节点规范化

适用对象：

- `stories/mygo.json`
- `stories/磁带主义story.legacy.json`
- 其他历史旧版故事文件

---

### 10.2 `js/runtime/formats/story-format-v2.js`

作用：

- 专门处理新版 V2 故事协议
- 负责：
  - `schemaId` 强识别支持
  - `content` token 规范化
  - V2 节点和图片字段规范化

---

### 10.3 `js/runtime/shared/story-format-detect.js`

作用：

- 统一识别故事文件是 V1 还是 V2
- 然后把原始 JSON 交给对应的格式模块处理

当前策略：

1. 先检查 `schemaId`
2. 再兼容旧的 `nodes[]` 型 V2
3. 最后回退到 V1

这是运行时和编辑器共用的协议入口。

---

### 10.4 `js/runtime/shared/story-runtime-core.js`

作用：

- 纯运行时核心逻辑
- 不处理页面特有 UI

负责内容：

- 构建 `nodeMap`
- 规范化 story 对象
- token -> 打字机队列转换
- 打字机控制器

这是“故事如何被播放”的底层核心。

---

### 10.5 `js/runtime/shared/story-runtime-ui.js`

作用：

- 共享运行时 UI 逻辑
- 面向 DOM 渲染

负责内容：

- 选项按钮渲染
- 节点切换显示
- 打字跳过
- 时钟更新
- 共享图片渲染器 `createTerminalImageRenderer`

说明：

- 现在三个运行页都使用同一套图片渲染逻辑
- 只要故事 JSON 中的 `image` 字段写的是有效路径，运行时就会直接加载

---

## 11. `js/pages/`：页面入口模块

### 11.1 `js/pages/runtime-open-local.js`

作用：

- 对应 `磁带主义-打开本地文件版本.html`
- 负责：
  - 探测默认故事文件
  - 手动文件选择
  - 恢复/重试提示
  - 启动共享运行时

说明：

- 默认故事路径是 `./stories/磁带主义story.json`

---

### 11.2 `js/pages/runtime-fetch.js`

作用：

- 对应 `磁带主义-读取本地文件版本.html`
- 负责：
  - 直接读取主故事 JSON
  - 启动共享运行时

说明：

- 默认故事路径是 `./stories/磁带主义story.json`

---

### 11.3 `js/pages/runtime-inline.js`

作用：

- 对应 `磁带未来主义交互小说页面.html`
- 负责：
  - 读取页面内联的 `window.__INLINE_STORY__`
  - 启动共享运行时

---

### 11.4 `js/pages/story-editor.js`

作用：

- 对应 `磁带主义剧本编辑器.html`
- 负责编辑器的全部交互逻辑

当前实现特点：

- 先调用共享规范化层读取故事
- 再把规范化后的故事投影成编辑器内部状态
- 导出时统一输出 V2 + `schemaId`
- 默认图片路径写成 `images/<filename>`

---

## 12. 运行关系总结

可以把项目理解为下面这套关系：

### 数据层
- `stories/磁带主义story.json`（V2 主故事）

### 协议层
- `story-format-v1.js`
- `story-format-v2.js`
- `story-format-detect.js`

### 运行核心层
- `story-runtime-core.js`
- `story-runtime-ui.js`

### 页面入口层
- `runtime-open-local.js`
- `runtime-fetch.js`
- `runtime-inline.js`
- `story-editor.js`

### 视觉层
- `css/runtime/index.css` 及其子样式文件

## 13. 如果要继续开发，通常改哪里？

### 想改故事内容
改：
- `stories/磁带主义story.json`

### 想改故事协议
改：
- `js/runtime/formats/story-format-v1.js`
- `js/runtime/formats/story-format-v2.js`
- `js/runtime/shared/story-format-detect.js`

### 想改播放行为
改：
- `js/runtime/shared/story-runtime-core.js`
- `js/runtime/shared/story-runtime-ui.js`

### 想改页面启动方式
改：
- `js/pages/runtime-open-local.js`
- `js/pages/runtime-fetch.js`
- `js/pages/runtime-inline.js`

### 想改三个运行页的统一视觉
改：
- `css/runtime/` 下对应样式文件

### 想改编辑器逻辑
改：
- `js/pages/story-editor.js`
- `磁带主义剧本编辑器.html`

### 想改故事/图片目录组织
改：
- `stories/`
- `images/`

## 14. 当前已知注意事项

1. 三个运行页已经共用一套 CSS 和一套运行时模块，不建议再回到内联重复实现。
2. V2 故事文件必须保留：
   - `schemaId: "cassette-if.story/v2"`
3. V1 文件不要在顶层增加元数据，否则旧版兼容逻辑会把它当节点读取。
4. 运行时不会猜测图片目录，而是直接使用 JSON 中 `image` 字段写入的路径。
5. 若某个 `images/...` 文件不存在，运行时会显示图片加载失败，这属于资源问题，不是协议问题。
